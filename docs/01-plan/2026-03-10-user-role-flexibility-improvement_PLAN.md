# PDCA Plan - 사용자 역할 설정 유연화 개선

## 1. 개요
사용자 생성(`createUser`) 시 역할을 `ROLE_USER`로 고정하던 로직을 개선하여, 입력값이 있을 경우 해당 역할을 부여하고 없을 경우에만 기본값을 사용하도록 한다.

## 2. 목표
- `UserInDto`에 포함된 `role` 값이 있다면 이를 우선적으로 적용한다.
- `role` 값이 `null`일 경우 기존처럼 `USER_ROLE.ROLE_USER`를 기본값으로 설정한다.
- 불필요하게 고정된 하드코딩 로직을 제거한다.
