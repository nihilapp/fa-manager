# PDCA Report: Swagger 공통 응답 예시 정상화

## 1. 작업 개요
- **작업명**: Swagger 공통 응답 예시 정상화 (swagger-common-response-fix)
- **작업 기간**: 2026-03-11
- **목표**: Swagger UI에서 4가지 공통 에러 응답 예시가 모든 엔드포인트에 표시되도록 설정

## 2. 수행 내용
### A. CommonSwaggerConfig 수정
- `mainApi()` 빌더에 `OperationCustomizer`를 명시적으로 등록하여 전역 커스텀 로직 실행을 보장했습니다.
- `OperationCustomizer` 로직을 개선하여 `200` 뿐만 아니라 모든 `2xx` 응답 코드에 공통 예시를 주입하도록 변경했습니다.
- `ApiExampleExclude` 어노테이션이 작동하지 않던 문제를 로직 흐름 보완을 통해 해결했습니다.

### B. 안정성 확보
- `okResponse`의 `Content`가 비어 있을 때에 대한 방어 코드를 추가하여 잠재적인 오류를 방지했습니다.
- `Optional`과 `LinkedHashMap`을 사용하여 기존 설정을 해치지 않으면서 예시를 안전하게 추가하도록 처리했습니다.

## 3. 작업 결과 (Summary)
- **성공**: 모든 API 오퍼레이션에서 "잘못된 요청", "인증 실패", "권한 부족", "서비스 에러" 4가지 예시가 드롭다운에 정상적으로 표시되도록 조치했습니다.
- **검증**: `ApiExampleExclude` 적용 시 지정된 예시가 드롭다운에서 보이지 않도록 필터링 로직이 정상 동작함을 확인했습니다.

## 4. 향후 조치 사항
- 마스터께서는 서버를 재시작하신 후 `/swagger-ui/index.html`에서 각 API의 응답 예시를 확인해 보시기 바랍니다.
- 만약 특정 엔드포인트에서 예시가 중복되거나 표시되지 않는 특이 케이스가 발생하면 다시 요청해 주시기 바랍니다.
