# PDCA Design - 응답 메시지 존댓말 변경

## 1. 개요
`RESPONSE_CODE.java`와 `response_message.yaml`의 메시지 변환 상세 설계.

## 2. 구현 상세

### 2.1 RESPONSE_CODE.java 메시지 변환 로직
- **성공(2xx)**: "~에 성공하였습니다", "~되었습니다" 등의 완성형 정중한 표현 사용.
- **클라이언트 오류(4xx)**: "~입니다", "~해 주세요", "확인해 주세요" 등의 정중한 요청 및 안내 표현 사용.
- **서버 오류(5xx)**: "~발행하였습니다", "~없습니다" 등의 명확하고 정중한 사실 전달 표현 사용.

### 2.2 response_message.yaml 메시지 변환 로직
- `default`, `health`, `auth`, `user` 섹션의 모든 메시지를 동일한 톤앤매너로 수정.
- 플레이스홀더(`{0}`)가 포함된 경우에도 정중한 문맥을 유지.

## 3. 예외 처리
- 기존 코드의 의미가 훼손되지 않도록 주의한다.
- 괄호 안의 HTTP 상태 코드는 유지한다.

## 4. 기대 결과
- 모든 API 응답 메시지가 정중한 존댓말로 반환되어 사용자 경험이 향상된다.
